# 日报技能回复模板

本文件用于统一查询类、提醒类请求的回复风格，减少同一种调用结果出现不同表达。

原则：

- 先给结论，再给细节
- 优先输出整理后的自然语言，不直接贴原始 JSON
- 人名、时间范围、结果状态必须明确
- 结果为空时直接说明，不绕弯

## 1. 查询日报

### 查某个时间范围的所有人日报

模板：

```text
以下是{时间范围}的日报情况：

{姓名A}
{日报正文或摘要}

{姓名B}
{日报正文或摘要}
```

简化模板：

```text
已查询{时间范围}日报，共 {人数} 人有日报。
```

### 查某个人的日报

模板：

```text
以下是{姓名}{时间范围}的日报：

{日报正文或摘要}
```

### 没查到日报

模板：

```text
未查询到{时间范围}{姓名可选}的日报。
```

## 2. 查询成员

### 返回成员列表

模板：

```text
当前需要写日报的成员共有 {总数} 人：
{姓名A}、{姓名B}、{姓名C}
```

### 没有可用成员

模板：

```text
当前没有查询到可用的日报成员。
```

## 3. 提醒日报

### 提醒成功

模板：

```text
已发送日报提醒。
成功提醒：{成功成员列表或人数}
```

### 部分成功

模板：

```text
已发送日报提醒。
成功：{sent}
跳过：{skipped}
失败：{errors}
```

### 无需提醒

模板：

```text
今天所有应写成员都已提交日报，无需提醒。
```

### 找不到可提醒成员

模板：

```text
没有找到可提醒的有效成员，因此未发送提醒。
```

## 4. 缺少关键信息时的追问

### 缺少时间范围

模板：

```text
你要查今天、本周、上周、本月还是上月的日报？
```

### 人名不明确

模板：

```text
你要查的是哪位成员的日报？请提供姓名。
```

## 5. 不推荐的回复方式

- 不要直接回整段接口 JSON
- 不要只说“执行成功”而不说明对象和结果
- 不要在提醒结果里泄露不必要的底层字段
