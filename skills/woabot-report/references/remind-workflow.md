# 日报提醒标准流程

本流程用于把“提醒没写日报的人”这类自然语言请求，稳定转换为固定的 API 调用链。

## 适用请求

- 提醒没写日报的人
- 催一下今天没交日报的人
- 帮我提醒还没发日报的同事
- 看看谁没写日报并提醒一下

## 固定执行顺序

1. 调用 `woa_daily_members`，获取全部应写日报成员
2. 调用 `woa_daily`，并固定使用 `{ "period": "today" }`
3. 从日报结果中提取已提交日报的人员 ID 或人员名
4. 用“应写成员 - 已提交成员”计算出待提醒成员
5. 若待提醒成员为空，直接结束，不调用提醒接口
6. 若待提醒成员非空，一次性调用 `woa_daily_remind`

## 推荐调用模板

### 第一步：查成员

```json
{}
```

### 第二步：查今日日报

```json
{ "period": "today" }
```

### 第三步：批量提醒

```json
{ "member_ids": ["uid_001", "uid_002"] }
```

自定义文案：

```json
{ "member_ids": ["uid_001", "uid_002"], "message": "请尽快提交今天的日报" }
```

## 强约束

- 不要跳过 `woa_daily_members`
- 不要根据昵称手写成员 ID
- 不要拆成多次提醒调用
- `message` 中不要写 `@`
- `message` 中不要写具体人名
- 默认按“今天”提醒，除非用户明确指定别的提醒语境

## 结束条件

- 没有人需要提醒：直接说明“今天所有应写成员都已提交日报”
- 有人需要提醒：返回 `sent`、`skipped`、`errors` 的整理结果
