# 日报查询示例

本文件用于收纳高频查询话术和推荐调用参数，降低自然语言到 API 参数映射时的波动。

## 时间映射

- 今天 -> `today`
- 本周 -> `this_week`
- 上周 -> `last_week`
- 本月 -> `this_month`
- 上月 -> `last_month`

## 高频示例

### 查今天所有人的日报

用户话术：

- 查今天日报
- 看看今天谁写了日报
- 拉一下今天的日报

推荐调用：

```json
{ "period": "today" }
```

### 查某人本周日报

用户话术：

- 查张三本周日报
- 看李四这周工作总结
- 王五这周更新了什么

推荐调用：

```json
{ "period": "this_week", "person": "张三" }
```

### 查某人上周日报

用户话术：

- 查张三上周日报
- 李四上周总结发了吗

推荐调用：

```json
{ "period": "last_week", "person": "张三" }
```

### 查本月所有人的日报

用户话术：

- 查本月日报
- 拉一下这个月的日报

推荐调用：

```json
{ "period": "this_month" }
```

### 用户没说时间范围

用户话术：

- 查日报
- 看一下日报
- 张三写日报了吗

处理建议：

- 如果上下文已有明确时间范围，则直接继承
- 如果上下文没有明确时间范围，则只追问一次时间范围

推荐追问：

```text
你要查今天、本周、上周、本月还是上月的日报？
```

## 输出建议

- 查询结果优先按人归纳
- 先展示姓名，再展示时间范围和正文摘要
- 用户没有要求原始结构时，不直接输出接口原始 JSON
